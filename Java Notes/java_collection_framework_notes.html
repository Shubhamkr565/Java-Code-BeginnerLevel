★ What is Collection Framework?
Definition: A set of classes & interfaces in Java that provide ready-made, reusable Data Structures to store and manage groups of objects — like a well-stocked toolbox for data storage!
📦 Container / Bag
Each collection class acts like a container that can hold multiple objects under one variable name.
🛠 Ready-made DSA
Backed by standard data structures — no need to write sorting, searching, or insertion logic from scratch.

★ Limitations of Array (Why We Need Collections)
1Fixed Size: Once declared, the size cannot increase or decrease. You're stuck with what you allocated.
2No Ready-made Methods: Even printing elements requires a manual for loop. No insert, delete, or search helpers. The only built-in variable is length.
3Homogeneous Data: Only same-type elements. Object[] is a workaround, but then you must downcast to access specific methods.
// Array of Student objects — fixed size! Student[] students = new Student[5]; students[0] = new Student(10, "S1", 780); // Object[] workaround for mixed types — but needs downcasting! Object[] objs = new Object[3]; objs[0] = new Student(); objs[1] = new Employee(); objs[2] = new Product(); ((Student) objs[0]).getMarks(); // ← must downcast every time
★ Features of Collection Framework
📈 Growable
Collections resize automatically. No need to specify or manage capacity manually.
🎭 Heterogeneous
Can hold both same-type and mixed-type elements in a single collection.
⚙️ Ready-made Methods
Built on standard DSA — add, remove, sort, search and more are all available out of the box.
📦 Objects Only
Primitives are not supported directly. Java uses autoboxing (int → Integer) automatically.
📌 Package Note: Almost all Collection classes live in java.util package.

★ Main Interfaces — When to Use Which?
List
java.util.List
  • Index-based ordering
  • Duplicates allowed ✓
  • Insertion order kept
  • Use: shopping cart, playlist
Set
java.util.Set
  • No duplicates allowed ✗
  • No guaranteed order (HashSet)
  • Sorted order (TreeSet)
  • Use: unique usernames, tags
Queue
java.util.Queue
  • FIFO order by default
  • Priority-based processing
  • peek / poll / offer methods
  • Use: task queue, print jobs
★ Collection Interface — The Root
Root Interface of the entire Collection Framework (except Map). Defines the common contract all collection classes must follow — like a master blueprint.
⚠ Important Note
No concrete class directly implements the Collection interface. It's always implemented through sub-interfaces like List, Set, or Queue.
✓ Common Methods Defined
add(), remove(), size(), contains(), isEmpty(), clear(), iterator() — usable across ALL collection types.
