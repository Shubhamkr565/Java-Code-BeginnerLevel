Polymorphic References — Creating ArrayList

An ArrayList object can be referenced by multiple types due to polymorphism — a child class object can be referred to by its parent type.

Which to use?

Best Practice: Always use List<String> list = new ArrayList<>();
Promotes loose coupling — swap ArrayList → LinkedList with zero other changes!
Use ArrayList reference only when you specifically need ensureCapacity() or trimToSize().

Code Examples

// Most recommended
List<String> list = new ArrayList<>();

// Only if needed: ArrayList-specific methods
ArrayList<String> al = new ArrayList<>();
al.ensureCapacity(100);

Creating Fixed Lists Quickly

Two utility methods let you create a List without manually calling add() repeatedly.

Arrays.asList() — Code

List<String> cities = Arrays.asList("Delhi", "Mumbai", "Chennai");
cities.set(1, "Hyderabad"); // ✓ OK
cities.add("Pune"); // ✗ UnsupportedOperationException!

// Backed by original array — changes sync both ways:
String[] arr = {"Apple", "Banana"};
List<String> fruits = Arrays.asList(arr);
arr[0] = "Guava"; // fruits[0] also becomes "Guava"!

List.of() — Code

List<String> cities = List.of("Delhi", "Mumbai", "Chennai");
// ALL modifications throw UnsupportedOperationException
// List.of("A", null, "B"); // ✗ NullPointerException!

String[] arr = {"Apple", "Banana"};
List<String> list = List.of(arr);
arr[0] = "Guava"; // list is unchanged — it's a copy
Need a modifiable list from fixed values?
List<String> list = new ArrayList<>(List.of("A","B","C"));

Primitive Array Gotcha!

int[] arr = {10, 20, 30};
List<int[]> list = List.of(arr); // size = 1 !! (whole array is ONE element)
Why? Generics only work with objects. An int[] is treated as a single Object. Use Integer[] instead.

Converting Collection → Array

Two overloaded toArray() methods from the Collection interface.

Method 1 — Object[] toArray()

Object[] arr = list.toArray();
for(Object obj : arr) {
String s = (String) obj; // manual cast needed
System.out.println(s.toUpperCase());
}

Method 2 — T[] toArray(T[])

// Best practice — pass new T[0]
String[] arr = list.toArray(new String[0]);
// JVM creates correct-sized array internally

// Also valid — pre-sized or larger array:
String[] arr2 = new String[list.size()];
list.toArray(arr2); // elements copied into arr2
Why new String[0]? Passing a zero-length hint is the modern best practice. The JVM sees the type info and allocates the right size array internally. Cleaner and avoids accidental stale elements.

Size Rules for T[] hint

  • Hint size == list size → elements copied into SAME array (no new allocation)
  • Hint size < list size → JVM creates a new array (your hint array is unused)
  • Hint size > list size → elements copied + trailing null at end

Key Methods & Time Complexity

Collection Interface Methods

List<String> cities = new ArrayList<>();
cities.add("Delhi"); cities.add("Mumbai");
cities.addAll(List.of("Hyderabad", "Bangalore"));
cities.contains("Delhi"); // → true
cities.remove("Mumbai"); // removes by object
cities.removeAll(toRemove); // removes all matching
cities.retainAll(toKeep); // keeps only matching

List Interface Methods

list.add(2, "Cherry"); // insert at index
list.get(3); // fetch by index
String old = list.set(4, "Pineapple"); // replace, returns old
String rem = list.remove(1); // remove by index
list.indexOf("Apple"); // first occurrence
list.lastIndexOf("Apple"); // last occurrence
List<String> sub = list.subList(1, 4); // view, toIndex exclusive
subList() returns a VIEW — modifying it affects the original list too!
