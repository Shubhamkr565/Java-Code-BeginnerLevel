Collections Class — Java

📦 java.util  ·  Utility Class  ·  All Static Methods  ·  Works with List, Set, Queue

What is Collections?
A built-in helper class with ready-made static operations — sort, search, shuffle, and more — that work on any List, Set, or Queue. No need to code the logic yourself!
⚠ Collections (utility class, has 's') ≠ Collection (interface, no 's')

🗺 Method Categories at a Glance:

Ordering Methods
sort() Sorts list in ascending (natural) order List<Integer> list = Arrays.asList(5, 2, 8, 1); Collections.sort(list); // → [1, 2, 5, 8] // Custom order with Comparator: Collections.sort(list, Comparator.reverseOrder());
reverse() Reverses the current order of all elements Collections.reverse(list); // → [8, 5, 2, 1]
shuffle() Randomly rearranges all elements Collections.shuffle(list); // → [2, 8, 1, 5] (random each time!)
Min / Max Methods
max() & min() Return the largest / smallest element in the list List<Integer> nums = Arrays.asList(3, 9, 1, 6, 4); int max = Collections.max(nums); // → 9 int min = Collections.min(nums); // → 1
Search Method
binarySearch() Finds the index of an element — ⚠ list MUST be sorted first! List<Integer> list = new ArrayList<>(Arrays.asList(1,3,5,7,9)); Collections.sort(list); // Must sort first! int idx = Collections.binarySearch(list, 7); // → index = 3
Modify Methods
fill() Replace ALL elements with a single given value (reset) List<Integer> list = new ArrayList<>(List.of(10,20,30,40,50)); Collections.fill(list, 0); // → [0, 0, 0, 0, 0]
swap() Swap two elements by their index positions List<Integer> list = new ArrayList<>(List.of(10,20,30,40,50)); Collections.swap(list, 0, 3); // → [40, 20, 30, 10, 50]
Info Methods
frequency() Count how many times an element appears List<String> list = Arrays.asList("A","B","A","C","A"); int count = Collections.frequency(list, "A"); // → 3
disjoint() Returns true if two collections share NO common elements List<Integer> a = List.of(10, 20, 30); List<Integer> b = List.of(70, 80, 90); System.out.println(Collections.disjoint(a, b)); // → true (zero elements in common)
Safety Methods
unmodifiableList() Makes list read-only — any write throws an exception List<String> list = new ArrayList<>(); list.add("A"); // ✓ allowed List<String> locked = Collections.unmodifiableList(list); locked.add("B"); // ⚠ UnsupportedOperationException!
synchronizedList() Wraps a list to make it safe for multi-threading List<Integer> safe = Collections.synchronizedList(new ArrayList<>()); // Thread A and Thread B can now use 'safe' safely!

✏ Student Practice Tasks

1 Create list [9, 3, 7, 1, 5]. Sort it → find max & min → then reverse it. Print after each step and explain the order of operations.
2 Sort the list from Task 1. Use binarySearch() to find element 7. Now try searching WITHOUT sorting first — what happens and why?
3 Create a list of student names with duplicates. Use frequency() to count how many times "Ali" appears. Then use fill() to reset all to "Unknown".
4 Create two lists and confirm they are disjoint. Add a common element to both — check disjoint again. Expected output: true then false.
5 Convert any list to an unmodifiableList. Try to call add, remove, and set on it. Which all throw exceptions? Can you still get or iterate?
