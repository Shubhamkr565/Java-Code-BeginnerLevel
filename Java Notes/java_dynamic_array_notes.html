★ What is an Array?

A fundamental data structure that stores multiple values of the same type under a single variable name. Each value is accessed directly by its index.

⚠ Limitations
  • Fixed size — cannot grow or shrink
  • Homogeneous — same data type only
  • Only 1 built-in variable: length
✓ Strengths
  • Direct O(1) access via index
  • Memory-contiguous (fast cache)
  • Simple & lightweight structure
int[] arr = {10, 20, 30, 40, 50}; // Fixed size = 5 arr[0] → 10, arr[4] → 50 // Direct access by index
★ Dynamic Array — The Problem it Solves
Problem: Normal arrays have fixed capacity. If you add too many elements, you run out of space!
Solution: A Dynamic Array automatically doubles its capacity when full — just like Java's ArrayList.
📊 How Array Resizing Works:
★ Key Code — MyDynamicArray.java
Core Concept: if (size == arr.length) → create newArr[arr.length * 2] → copy → arr = newArr
public void add(int data) { if (size == arr.length) { // Array is full? int[] newArr = new int[arr.length * 2]; // Double size for (int i = 0; i < size; i++) newArr[i] = arr[i]; // Copy old data arr = newArr; // Switch reference } arr[size++] = data; // Add element }
★ Methods at a Glance
add(int data)
Adds element; doubles array if full
remove(int index)
Shifts elements left; decrements size
get(int index)
Returns element; throws if out of bounds
size() / capacity()
Returns used slots / total array length
clear()
Resets size to 0 (data stays in memory)
isEmpty() / display()
Checks if empty / prints all elements
Remember: Java's ArrayList works exactly this way internally — it starts small and grows by ~50% each time. That's why inserting at the end is amortized O(1)!
